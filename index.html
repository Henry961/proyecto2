Empezamos con el archivo plano

Un archivo plano es un conjunto de datos almacenados en un dispositivo de almacenamiento, en formato texto sin procesar. Está compuesto por líneas de texto separadas entre sí por caracteres como el salto de línea o espacios.

Sirve para:

  • Intercambio de datos entre diferentes aplicaciones
  • Almacenamiento temporal de datos
  • Configuraciones predeterminadas para programas e impresoras

características clave

  • Uso sencillo
  • Flexibilidad
  • Búsquedas rápidas
  • Facilidad para exportar e importar datos
  • Asequible

Tipos de archivos planos

  • Archivo CSV
  • Archivo INI
  • Archivo TXT
  • Archivo XML

Archivo XML

El lenguaje de marcado extensible (XML) permite definir y almacenar datos de forma compartible. XML admite el intercambio de información entre sistemas de computación, como sitios web, bases de datos y aplicaciones de terceros. Las reglas predefinidas facilitan la transmisión de datos como archivos XML a través de cualquier red

beneficios de usar XML

  • Respaldo para las transacciones interempresariales
  • Conservación de la integridad de los datos
  • Mejora de la eficiencia de búsqueda
  • Diseño de aplicaciones flexibles

aplicaciones de XML

  • Transferencia de datos
  • Aplicaciones web
  • Documentación
  • Tipo de datos

componentes de un archivo XML

  • Documento XML
  • Declaración XML
  • Elementos XML
  • Atributos XML
  • Contenido XML

Archivo JSON

Los archivos tipo json (JavaScript Object Notation), es un formato el cual es implementado por Java. Hace parte de uno de los lenguajes más importantes de la programación y tiene como fin la creación de páginas web como también en la generación de múltiples software.

características de JSON

  • Lenguaje independiente
  • Formatación del archivo JSON:
    • Llaves { }
    • Corchetes [ ]
    • Dos puntos :
    • Coma ,

beneficios del formato JSON

  • Lectura más sencilla
  • Más agilidad en la ejecución y el transporte de datos
  • Archivos más livianos
  • Parsing más fácil

Archivo Parquet

Apache Parquet es un formato de almacenamiento en columnas que proporciona optimizaciones para acelerar las consultas, es un formato de código abierto (open source) que ofrece alternativas de almacenamiento, codificación, compresión y lenguajes de programación, entre otras.

Esta compuesto por:

  • Row Group
  • Column Chunk
  • Page

se admite para los conectores siguientes

  • Amazon S3
  • Almacenamiento compatible con Amazon S3
  • Azure Blob
  • Azure Data Lake Storage Gen1
  • Azure Data Lake Storage Gen2
  • Archivos de Azure
  • Sistema de archivos
  • FTP
  • Google Cloud Storage
